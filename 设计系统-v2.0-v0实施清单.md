**阴影 Token（v2.3 新增 · 给卡片纵深感）**

**使用规则（很重要，按此执行）**

- **所有按钮 / Tag / chip 强制胶囊形**（`border-radius: var(--r-pill)`）—— 这是 V2 视觉签名
- 所有卡片至少 `var(--r-card)` 圆角，禁止 90° 直角
- 所有卡片至少 `var(--shadow-sm)` 阴影，禁止"贴在背景上的平面感"
- 一个屏幕里 `--rust` / `--amber` 加起来出现不超过 3 次。多了就变咨询报告
- 卡片**默认无边框**，靠空间和背景色区分。需要边框时只用 `--line`，1px
- `--moss-deep` 只用在两个地方：pull quote 反色块、Manifesto-style 大段强调
- 禁止使用纯黑 `#000` 或纯白 `#FFF`
- 禁止额外引入紫色、粉色、亮蓝色等任何饱和度高的颜色

### 1.2 字体 Token

**字体栈（从 Google Fonts 引入）**

```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>

```

**Token 定义**

```css
:root {
  --font-display: "Fraunces", "Noto Serif SC", "Songti SC", serif;
  --font-body: "Inter", "PingFang SC", -apple-system, system-ui, sans-serif;
  --font-mono: "JetBrains Mono", ui-monospace, monospace;
}
```

**字号阶梯（Modular Scale，移动端用 clamp 自适应）**

| 用途 | 字号 | 行高 | 字距 | 字重/样式 |
|---|---|---|---|---|
| Hero h1 | `clamp(32px, 4vw, 52px)` | 1.2 | -0.01em | **500 (v2.3 调轻)**, display |
| Pull quote / Manifesto Lite | `clamp(28px, 3.4vw, 40px)` | 1.4 | -0.01em | 400 italic, display |
| Section h2 | `clamp(28px, 3.4vw, 40px)` | 1.2 | -0.01em | 600, display |
| Card h3 | `20-22px` | 1.3 | 0 | 600, body |
| Lead 段 | `17px` | 1.65 | 0 | 400, body |
| Body | `15-16px` | 1.7 | 0 | 400, body |
| Small | `13-14px` | 1.5 | 0 | 400, body |
| Eyebrow | `11px` | 1 | 0.16em | 700 UPPERCASE, body |
| Stat 大数字 | `clamp(40px, 5vw, 64px)` | 1.0 | -0.02em | 600, display, 老式数字 (`font-feature-settings: 'onum'`) |

> **v2.3 字号下调说明：** 第一版上线后用户反馈字号太大、设计感不足。v2.3 把 Hero h1 从 56-88px 下调到 32-52px、字重从 700 改到 500，给 italic 强调留空间，避免标题"压垮"页面。

**强调字规则（v2.3 强化）**

- 中文里需要强调的词（如 Hero 的"AI 行业研究"）**必须** `font-style: italic` + `color: var(--moss-deep)`
- 英文标题里的强调词**必须**用 Fraunces italic（如 "_taste_"）
- 不允许只用 `font-weight: 700` 做强调——那是工程师的强调方式，不是杂志的

**字体使用原则**

- 中英混排时，英文用 Fraunces（display）/ Inter（body），中文用 Noto Serif SC（display）/ PingFang SC（body）。CSS `font-family` 直接堆叠交给浏览器自动切换
- 任何"标题级"内容用 display；正文、按钮、表单用 body
- italic **只用在 display 字体上**（Fraunces italic 极美），body 字体禁止 italic
- 全页面**没有任何字号小于 11px**

- `max-width: 1200px` —— 整体内容容器

### 1.5 全局动效

> **v2.1 用户决策：** 全局光标光斑保留——你说有现成源码可以直接喂给 AI，效率高且能让网站精致一点。禁止清单收紧到只保留"重型滚动视差" + "影响可读性的动画"两条硬红线。

**v0 要做的事**

1. **卡片 hover**：transform `translateY(-2px)` + 阴影微涨，过渡 200ms `cubic-bezier(0.2, 0.8, 0.2, 1)`
2. **进场淡入**：所有 section 滚到视口时 fadeUp 16px + opacity 0→1，600ms
3. **锚点跳转**：CSS `scroll-behavior: smooth`
4. **文章详情页 sticky TOC**：右侧目录跟随阅读位置高亮（仅文章页用）
5. **全局光斑（鼠标跟随）**：跟随光标的暖琥珀光晕

**全局光斑实现示例（可直接喂给 AI，或用你收藏的动效网站源码替换）**

```js
const halo = document.createElement('div');
halo.style.cssText = `
  position: fixed;
  width: 600px;
  height: 600px;
  border-radius: 50%;
  background: radial-gradient(circle, rgba(212, 165, 116, 0.16), transparent 60%);
  pointer-events: none;
  z-index: 0;
  transform: translate(-50%, -50%);
  transition: transform 400ms cubic-bezier(0.2, 0.8, 0.2, 1);
  mix-blend-mode: multiply;
`;
document.body.appendChild(halo);
window.addEventListener('mousemove', e => {
  halo.style.transform = `translate(${e.clientX}px, ${e.clientY}px) translate(-50%, -50%)`;
});
```

调参说明：饱和度通过 `rgba(212, 165, 116, X)` 的第 4 位调；尺寸通过 width/height 调；速度通过 transition 时长调。也可以换成苔藓绿 `rgba(93, 112, 82, X)`。

**结尾签名（P2 加分项，v0 不做）**

- 位置：Footer 上方居中
- 内容：SVG 手写"宁"字
- 实现：`stroke-dasharray` + `stroke-dashoffset` + IntersectionObserver
- 笔画颜色 `--moss-deep`，宽度 2-3px

**禁止清单（v2.1 收紧版）**

- ❌ 重型滚动视差（影响性能）
- ❌ 任何影响可读性 / 干扰阅读流的动画

**允许但需克制使用**

- 鼠标跟随光斑（已加入 v0 必做）
- 简单 3D（如果有非常具体的应用场景再加，比如某个卡片的微 3D 翻转）
- 转场动画（如果跟内容叙事有关，可以加）

### 6.4 重要：怎么处理 AI 跑偏

如果 AI 提出"我建议改成 XXX"或"另一个方案是 YYY"——

- 第一反应：检查文档里有没有规定。**有规定就按文档**，AI 的建议忽略
- 如果文档里没规定：让 AI 列出 2-3 个选项 + 各自的 trade-off，由你决定，**不要让 AI 替你选**
- 如果文档里规定有问题：先记到一个 `设计系统-v2.0-勘误.md` 文件里，**不要直接改主文档**，避免文档漂移
