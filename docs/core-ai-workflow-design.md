# 核心 AI 工作流 · 定稿说明（招聘向）

**受众**：招聘方  
**目标**：同时证明 **A** 从 0 到可交付的链路，与 **C** 多工具编排能力。  
**与九件套关系**：图中仅出现本链路用到的工具子集；工具标签一律 **英文原名**。上方 skills 区负责「有什么」，本区负责「**这条链路怎么跑通**」。

---

## 读图语法（两张卡一致）

1. 时间轴 **从左 → 右**，箭头样式统一。  
2. 每个阶段块内最多三行信息：  
   - **阶段名**（中文短，2～4 字）  
   - **产出物**（中文，一行）  
   - **Tools**（英文原名小标签；该阶段最多 2 个主工具，更多用 `+` 或并入文案）  
3. **Outcome 缩略图**：每张卡用一处小图锚定「最终交付」——Case 01 用本页 Hero / 原型一屏；Case 02 用公众号文章首屏（可后换真实素材）。  
4. **底部 chips** 与图中出现的工具 **一致**，不新增未在图中出现的工具名。

---

## Card 01 · Vibe Coding / 个人网站（Case 01）

**阶段（5 段，可压 4 若版面紧；当前按 5 定稿）**

| 顺序 | 阶段名（示意） | 产出物（中文一行） | Tools（英文原名） |
|------|----------------|-------------------|-------------------|
| 1 | Brief | 目标 / 受众 / 成功标准 | `Cursor` |
| 2 | Spec | 需求与结构包（brief → PRD 感） | `spec-generator` |
| 3 | Design | 视觉与组件基调（tokens / 参考） | `frontend-design` |

**Outcome**：链左侧或上方放 **本页一屏截图**，箭头指向阶段 1，表示「从目标到当前可见页面」。

**底部 chips（与图一致）**：`spec-generator` · `frontend-design` · `Cursor` · `Claude Code`
